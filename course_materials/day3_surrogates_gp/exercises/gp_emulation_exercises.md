## Exercise 1: Sampling from the Prior

Before we can condition a Gaussian process on observations, we must understand how our choice of prior distribution defines the properties of the functions we expect to see. A Gaussian process prior is completely defined by its mean function (here assumed to be zero) and its covariance function (the kernel).

### Mathematical Background
A Gaussian process prior $f(x) \sim \mathcal{GP}(0, k(x, x'))$ implies that any finite collection of points evaluated on this function follows a multivariate normal distribution:
$$\mathbf{y}_* \sim \mathcal{N}(\mathbf{0}, K_{**})$$

Where $K_{**}$ is the $N \times N$ covariance matrix generated by evaluating the kernel function at every pair of test locations:
$$[K_{**}]_{i,j} = k(x_{*,i}, x_{*,j})$$

The most ubiquitous covariance structure is the **Squared Exponential (SE)** or **Radial Basis Function (RBF)** kernel:
$$k(x, x') = \sigma^2 \exp\left( -\frac{(x - x')^2}{2\ell^2} \right)$$

* $\sigma^2$ is the **output variance**, which governs the vertical scale (amplitude) of the functions.
* $\ell$ is the **lengthscale**, which dictates how fast the correlation decays over horizontal distance, directly determining the "smoothness" or frequency of the function variations.

It is important to understand what the possible family of functions might look like. We sometimes will call this *prior predictive checking*. This is a process of ensuring that the sort of outputs we are seeing from our prior assumptions lead to reasonable functions which we expect to see in our data.

### Tasks
1. **Define the Test Grid:** Create a vector of $N = 100$ evenly spaced test points, $X_*$, spanning the interval $[0, 5]$.
2. **Generate Covariance Matrices:** Compute $K_{**} = k(X_*, X_*)$ using baseline hyperparameters: $\ell = 1.0$ and $\sigma^2 = 1.0$. You can also look at the structure of this covariance matrix for your test grid, for example by doing `plt.matshow(kss)`.
3. **Numerical Stability Correction (The Jitter):** Due to finite-precision arithmetic, large covariance matrices can lose positive-definiteness, even though this is guaranteed by the kernel form. To fix this, we can sdd a small "jitter"/"nugget" term to the diagonal before sampling:
   $$K_{**} \leftarrow K_{**} + 10^{-6}\mathbb{I}_N$$
4. **Draw Samples:** Extract 5 random function samples from $\mathcal{N}(\mathbf{0}, K_{**})$. 
   * We implement the sampling via Cholesky decomposition (sometimes called a *whitening transform*): compute $L = \text{cholesky}(K_{**})$, draw a matrix of independent standard normals $Z \sim \mathcal{N}(0, 1)$ of shape $(N, 5)$, and compute your samples as $\mathbf{Y}_* = L Z$.
5. **Visualize and Experiment:** Plot your 5 samples against $X_*$. Re-run your script with differing $\ell$ values. Take note of how the physical structure of the functions changes with $\ell$.
6. **Kernel Compositions** Remember that we have also seen in the notebook that we get valid kernels by summing or multiplying existing kernels (elementwise if we have the matrices). Try to have a look at what function draws from different combinations of kernels look like. A word of warning, as we combine kernels we can become more expressive but it can become less intuitive to set our hyperparameters *a priori*.

## Exercise 4: Dataset Selection 

In engineering it is often very expensive to acquire our training data pairs. We have focussed on when each datapoint is the result of a *numerical simulator* which could be computationally very expensive - e.g. I may need to run an FEM/CFD model which takes many days or I might need to run a physical experiment taking months. Therefore, we might want to think "what data points will be best to improve my model?". 

This will clearly depend on what we are using the model for and in some circumstances we might not purely want to focus on the prediction performace across the whole domain. We might want to improve our ability to find the minimum/maximum of the function we are fitting (this is the basis of Bayesian Optimisation) or we might have other quantities of interest.

We can look at a very simple way of implement this data point choice loop. We are going to look here at choosing the *best* points from our available CHF data - this is the *pool-based active learning* setting.

The approach we show in the notebook is the most straightforward method. 
1. Initalise the GP with some small random set of data
2. Predict all remaining possible training points
3. Choose the one with the highest variance to add into the training set
4. Loop until golden brown

Beware, this loop to update the GP is quite slowly implemented. You may want to limit the number of training examples. There are more efficient ways to do this approach, e.g. rank-one Cholesky updates if you have fixed hyperparameters. 

### Tasks

4. If we knew there were particular pressure ranges which were more important than others where we wanted out emulator to work best, how might this affect our active learner?
